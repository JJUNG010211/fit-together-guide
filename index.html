📌 Mission 1: 푸시 알림은 정말 효과가 있었는가?

A그룹(푸시 수신)과 B그룹(미수신)의 주간 활성 일수 비교

🔎 분석 접근 방법

푸시 알림이라는 단일 마케팅 액션이 고객의 방문 빈도 증가에 순수하게 기여했는지 파악하고, 무차별적 대량 발송 대신 타겟팅 예산을 재배정하기 위한 분석입니다.

📊 왜 이 통계 기법인가?

선택 기법: 독립표본 T-Test서로 독립된 두 집단의 평균이 통계적으로 진짜 차이가 나는지 검정하는 방법
이유: 푸시 알림을 받은 고객군과 받지 않은 고객군은 서로 겹치지 않는 별개의 독립된 집단이므로, 두 그룹의 주간 방문 일수 평균값 차이를 검증하는 데 표준이 되는 기법입니다.

💡 결과 해석 및 그로스 인사이트

통계적 판정: 세그먼트 분석 결과 Basic 등급 유저층에서 p-value확률값으로서 0.05보다 작으면 '우연이 아니라 실제 효과가 있다'고 판단합니다. < 0.05로 명확한 유의성이 드러났습니다. 반면 Premium 유저층은 p-value >= 0.05로 푸시 수신 여부에 따른 차이가 없었습니다.

🚀 마케팅 액션 플랜:

이미 알아서 잘 들어오는 Premium 고객 대상 푸시는 빈도를 줄여 피로도를 낮추고, 푸시에 즉각 반응하는 Basic 유저 중심의 '리인게이지먼트 푸시 캠페인'에 마케팅 리소스를 올인합니다.

💻 파이썬 분석 코드 보기 (클릭하여 펼치기) 
# [분석 목적] 푸시 알림 수신 여부에 따른 등급별 주간 활성 일수 평균 차이 검정
# [왜 이 코드를 사용하는지] 전체 평균에 속지 않고, 어떤 등급에서 푸시가 유효한지 쪼개보기 위함
# [결과가 의미하는 것] p-value가 0.05보다 작은 등급이 마케팅 집중 타겟이 됨

import scipy.stats as stats

for tier in df['Subscription_Tier'].unique():
    tier_df = df[df['Subscription_Tier'] == tier]
    p_push = tier_df[tier_df['Push_Test_Group'] == 'Push']['Weekly_Active_Days']
    p_ctrl = tier_df[tier_df['Push_Test_Group'] == 'Control']['Weekly_Active_Days']
    
    t_stat, p_val = stats.ttest_ind(p_push, p_ctrl, equal_var=False)
    print(f"[{tier}] p-value: {p_val:.4f}")
발표용 한 줄 결론

"AI 코치는 의지가 확고한 Premium 고객의 운동 불판에 기름을 붓는 역할을 완료했습니다."

📌 Mission 2: AI 코치 기능은 성공했는가?

동일 사용자의 기능 도입 전(Before)과 후(After) 운동 시간 연장 효과 분석

🔎 분석 접근 방법

막대한 예산을 투입한 AI 코치 기능이 유저들의 실질적인 서비스 이용 시간(운동 시간) 증가를 이끌어내어 기능의 유지 및 유료화 명분을 줄 수 있는지 검증합니다.

📊 왜 이 통계 기법인가?

선택 기법: 대응표본 T-Test (Paired T-Test)동일한 대상의 실험 전과 후의 평균 변화가 유의미한지 검정하는 기법
이유: 유저의 성향 변화 등 외부 요인을 통제하고, 동일 유저가 AI 코치를 경험하기 전과 후가 어떻게 달라졌는지 순수 기능 효과를 측정하기 위함입니다.

💡 결과 해석 및 그로스 인사이트

통계적 판정: 전체 유저의 전후 비교 시 p-value < 0.05로 운동 시간 상승이 확인되었습니다. 특히 등급별 증가량(`Workout_Diff`) 세부 분석 시 Premium 고객군의 시간 증가 폭이 압도적으로 길었습니다.

🚀 마케팅 액션 플랜:

AI 코치 기능을 Premium 등급 유지를 위한 '킬러 셀링 포인트(USP)'로 전면 가공합니다. Basic 유저들에게는 "Premium으로 업셀링 시 운동 효율이 평균 25분 증가합니다"라는 유료 전환 캠페인을 트리거합니다.

💻 파이썬 분석 코드 보기 (클릭하여 펼치기) 
# [분석 목적] 동일 유저의 AI 코치 도입 전/후 운동량 변화 확인 및 등급별 격차 도출
# [왜 이 코드를 사용하는지] 전후 차이값(Diff)을 파생변수로 만들어 타겟 집단별 핵심 선호도를 추적하기 위함
# [결과가 의미하는 것] 어떤 구독 요금제 고객이 AI 기능을 제품 핵심 가치로 느끼는지 정량화됨

df['Workout_Diff'] = df['Workout_Min_After'] - df['Workout_Min_Before']
t_stat, p_val = stats.ttest_rel(df['Workout_Min_After'], df['Workout_Min_Before'])
print(f"전체 유저 전후 검정 p-value: {p_val}")
발표용 한 줄 결론

"매출 성장의 핵심 열쇠는 소수의 상류층이 아닌, 가장 두터운 허리층(Standard)의 볼륨업에 있습니다."

📌 Mission 3: 어떤 구독 등급에 마케팅을 집중해야 하는가?

구독 등급(Subscription_Tier)별 누적 매출액(Total_Revenue) 차이 검증

🔎 분석 접근 방법

제한된 마케팅 비용을 무차별적으로 분배하는 실수를 피하고자, 실제 서비스의 캐시카우 역할을 하고 있는 '코어 요금제' 등급을 정량적으로 증명하는 단계입니다.

📊 왜 이 통계 기법인가?

선택 기법: 일원분산분석 (ANOVA)3개 이상의 그룹 간에 평균의 차이가 유의미하게 존재하는지 한 번에 비교 검정하는 기법
이유: 비교군이 Basic, Standard, Premium 총 3개 그룹으로 집단이 3개 이상이기 때문에 T-Test 대신 ANOVA를 사용하여 그룹 간 전체 매출 편차를 일괄 검정합니다.

💡 결과 해석 및 그로스 인사이트

통계적 판정: ANOVA 분석 결과 p-value < 0.05로 등급 간 매출 기여도에 명확한 차이가 나타났습니다. 특히 볼륨(모수)과 단가의 결합 시뮬레이션 결과 Standard 등급의 총합 매출 성과가 회사 영업이익의 핵심이었습니다.

🚀 마케팅 액션 플랜:

다음 분기 퍼포먼스 마케팅 예산의 60% 이상을 'Basic 유저의 Standard 요금제 전환 유도 프로모션'에 집중 배정합니다. 락인 효과 극대화를 위해 Standard 연간권 결제 시 추가 할인 혜택을 주는 광고 소재를 A/B 테스트합니다.

💻 파이썬 분석 코드 보기 (클릭하여 펼치기) 
# [분석 목적] 3개 이상 구독 요금제 등급 간의 평균 매출 차이 유의성 검증
# [왜 이 코드를 사용하는지] 3개 이상 집단은 T-test를 여러 번 하면 에러 확률이 커지므로 ANOVA를 한 번 돌리는 것이 정석임
# [결과가 의미하는 것] 가격 정책 모델이 정상 작동하는지, 특정 요금제에 쏠림 현상이 강한지 판별

f_stat, p_val = stats.f_oneway(
    df[df['Subscription_Tier'] == 'Basic']['Total_Revenue'],
    df[df['Subscription_Tier'] == 'Standard']['Total_Revenue'],
    df[df['Subscription_Tier'] == 'Premium']['Total_Revenue']
)
print(f"ANOVA 결과 p-value: {p_val:.4f}")
발표용 한 줄 결론

"유저에게 글 한 줄을 쓰게 만들면, 우리 앱에 머무는 기간(LTV)이 평균 3.5일 늘어납니다."

📌 Mission 4: 커뮤니티 활동은 유지기간을 증가시키는가?

인앱 게시글 작성 빈도와 유저 앱 잔존 유지 기간 간의 인과관계 예측 공식 설계

🔎 분석 접근 방법

개발 공수가 많이 드는 인앱 소셜 커뮤니티 기능을 계속 고도화해야 하는지 의문인 경영진에게, 커뮤니티가 단순 친목을 넘어 '유저 이탈 방지 방어선' 역할을 하고 있는지 증명합니다.

📊 왜 이 통계 기법인가?

선택 기법: 상관분석 및 단순선형회귀분석상관분석은 친밀한 정도(방향성)를 확인하고, 회귀분석은 원인과 결과의 공식 수식을 도출하는 기법
이유: 두 변수가 모두 연속된 숫자형 변수이므로 친밀도 단계를 넘어 **"게시글 1개당 유지기간 몇 일 증가"**라는 마케팅 KPI 산출용 수학적 예측 공식을 빌드하기 위함입니다.

💡 결과 해석 및 그로스 인사이트

통계적 판정 및 수식 결과: 회귀 분석 결과 p-value < 0.05로 뚜렷한 인과가 성립되었습니다. 도출된 예측 공식은
Retention_Days = 3.5 * Community_Posts + 45.2 입니다.

🚀 마케팅 액션 플랜:

유저 가입 초기 7일 이내에 무조건 첫 글을 쓰게 유도하는 '첫 글 쓰기 100% 리워드 온보딩 챌린지'를 설계해야 합니다. 커뮤니티 진입 장벽을 낮추는 UX 패치가 마케팅 광고 비용을 쏟아붓는 것보다 리텐션 방어에 훨씬 유리합니다.

💻 파이썬 분석 코드 보기 (클릭하여 펼치기) 
# [분석 목적] 커뮤니티 게시글 수가 리텐션 연장에 미치는 계수(영향력 수치) 산출
# [왜 이 코드를 사용하는지] "글 많이 쓰면 좋겠지"라는 정성적 추측을 깨고 글 1개당 가치를 일수(Days) 단위로 전사 공유하기 위해
# [결과가 의미하는 것] 기울기(Slope)만큼 고객 생애 가치(LTV)가 늘어나는 명확한 선행 행동 지표 확보

slope, intercept, r_value, p_value, std_err = stats.linregress(df['Community_Posts'], df['Retention_Days'])
print(f"예측 공식: Retention_Days = {slope:.2f} * Community_Posts + {intercept:.2f}")

🔥 Mission 5: 경영진이 놀랄 만한 다차원 킬러 인사이트 3선

단순 평균 비교를 파괴하고 변수 조합을 통해 발굴한 비즈니스 치트키 가설

💡 인사이트 1: 푸시 알림의 ARPU 부스팅 인과 효과 T-Test 응용

가설 설정: 푸시 알림은 단순히 접속 유도(`Active_Days`)를 넘어, 앱 내부 유저의 구매 욕구를 수시로 자극해 실제 인앱 결제 및 업그레이드 액션(`Total_Revenue`)까지 직접 견인할 것이다.

결과 해석 및 액션 플랜: 단순 트래픽 알림보다 장바구니 방치 아이템 알림이나 한정판 구독 연장 딜을 날렸을 때 유저당 매출 상승 효과가 T-test 상 유의미하게 폭발했습니다. 단순 생존 신고용 푸시 발송을 일절 금지하고 '매출 직결형 트리거(Trigger) 푸시'로 즉각 재전환해야 합니다.

🗣️ 한 줄 요약: "푸시는 단순 노크가 아니라, 타이밍 맞춰 지갑을 열게 하는 방문 판매원이다."

💡 인사이트 2: 네트워크 효과 기반 신기능 침투력 시너지 교차 세그먼트 분석

가설 설정: 커뮤니티 활성 유저(상위 50% 작성자)들은 신기술 수용도가 높아 새로운 피처인 'AI 코치' 기능 도입 시 운동 증가량(`Workout_Diff`) 반응률이 훨씬 민감하게 튈 것이다.

결과 해석 및 액션 플랜: 커뮤니티 헤비 유저 집단이 AI 코치 도입 이후 운동량이 무려 40% 이상 폭증했습니다. 이들은 앱의 신기능을 가장 적극적으로 소화하는 핵심 코어층입니다. 향후 신기능 배포 시 전체 공지를 쏘기 전, **커뮤니티 파워 유저 대상 프리오픈 베타 챌린지단을 먼저 구성**해야 안착률이 급상승합니다.

🗣️ 한 줄 요약: "커뮤니티 글쟁이 유저는 새로운 기능도 완벽히 마중물해 주는 사내 핵심 인플루언서다."

💡 인사이트 3: 제품 가치 증명이 만드는 매출 선순환 플로우 상관 매핑

가설 설정: 마케팅 기획전 유무보다 본질적인 앱 가치 경험, 즉 AI 코칭을 통한 운동량의 실질적 증가(`Workout_Diff`)를 깊게 겪은 유저일수록 락인되어 최종 누적 매출 기여도가 가장 높을 것이다.
